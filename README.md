# HyoExam

시험(모의고사·내신) 시간표와 실시간 시계를 TV/모니터로 송출하는 Windows 앱.

- 실시간 시계는 표준 SNTP(UDP/123)로 10초마다 동기화합니다. 시간 기준은
  툴바 드롭다운에서 3곳 중 선택: **네이버 시계**(time.navyism.com),
  **KRISS**(한국표준과학연구원, ntp.kriss.re.kr), **KISA**(한국인터넷진흥원,
  time.bora.net).
- 순수 Win32 + Direct2D/DirectWrite — 프레임워크 오버헤드 없이 실행 파일
  1개(수 MB), 시작 즉시 렌더링, GC 없음.

## 조작

기본은 **창 모드**로 시작합니다(관리자 화면). `F11`로 전체화면(학생용 TV/
프로젝터 송출 화면)으로 전환하면 툴바와 저작권 표시가 전부 숨겨지고 시계·
시간표·메모만 남습니다. 창 모드 상단에는 전역 툴바가 있습니다: **왼쪽**에
시험 유형(모의고사/지필평가) 선택 버튼과 시간 기준 드롭다운, **오른쪽**에
저장/불러오기 아이콘, 테마 토글(해/달 아이콘), 전체화면 아이콘. 설정 모달은
따로 없고, 모든 옵션이 툴바에 바로 노출되어 있습니다.

| 동작 | 방법 |
|---|---|
| 창 모드 ↔ 전체화면 | `F11`, 또는 상단 툴바 오른쪽 끝 아이콘 (전체화면 중에는 좌측 상단의
  작은 복귀 아이콘으로도 나갈 수 있음 — 키보드 없는 환경 대비) |
| 전체화면 나가기 | `Esc` (또는 `F11`) — 전체화면으로 들어가면 편집 팝업은 자동으로 닫힘 |
| 시험 유형 전환 | 툴바 왼쪽의 모의고사/지필평가 버튼 |
| 시간 기준 전환 | 툴바의 드롭다운 (네이버 시계/KRISS/KISA) — 각 항목에 마우스를
  올리면 간략한 설명이 뜸 |
| 테마 전환 | 툴바의 해/달 아이콘 (라이트 ↔ 다크). 기본값은 **시스템 설정을 따라감** |
| 설정 저장/불러오기 | 툴바의 저장/불러오기 아이콘 — 이름만 입력해 현재
  테마·시간 기준·시간표 전체를 한 번에 저장, 목록에서 클릭 한 번으로
  복원. 저장하면 "저장되었습니다" 안내가 잠깐 떴다가 사라짐 |
| 홈페이지 방문 | 창 모드 하단 우측의 `hyot.dev` 텍스트(옆 링크 아이콘) 클릭 →
  기본 브라우저 새 창으로 열림 |

좌(시계 영역)/우(시간표) 패널 비율은 TV·프로젝터 송출에 맞춘 6:4로 고정되어
있습니다 (드래그로 조절하던 이전 방식은 제거됨). 좌측 시계 영역은 다시 위(시계,
7)/아래(메모, 3) 비율로 나뉩니다.

### 메모 (좌측 하단, 창/전체화면 모두 표시)

시계 바로 아래에 자유롭게 글을 쓸 수 있는 메모 영역이 있습니다. 창 모드에서만
편집 가능하고(전체화면에서는 읽기 전용으로 표시), 내용은 서식 그대로
`%APPDATA%\HyoExam\memo.rtf`에 자동 저장됩니다.

- 클릭 후 바로 타이핑, 마우스 드래그로 원하는 부분만 블록 선택
- 툴바: **B**(굵게) · **U**(밑줄) · **A-/A+**(글자 크기) · **가**(글자색) ·
  **■**(배경색) — 모두 선택 영역에만 적용되고, 선택이 없으면 커서 위치부터
  새로 입력하는 글자에 적용됨
- 글자색/배경색은 Windows 기본 색상 선택 대화상자 사용

### 시간표 편집 (창 모드에서 항상 가능 — 별도 켜기 불필요)

- **교시 추가/수정**: 우측 카드의 교시 행에 마우스를 올리면 살짝 떠오르는
  느낌으로 강조되어 클릭 가능함을 알려줍니다. 클릭하면 팝업에서 교시명·과목·
  시작·종료(시:분 드롭다운으로 선택) 입력 후 저장 (소요시간은 자동 계산).
  각 행의 ✕ 아이콘으로 즉시 삭제, `+ 교시 추가` 버튼으로 새 교시 추가
- **교시 순서 변경**: 행을 누른 채로 위/아래로 드래그하면 다른 행들이 즉시
  자리를 비켜주며 순서가 바뀝니다 — 놓으면 그 순서로 저장
- 모든 편집은 즉시 `data/schedules.json`에 저장됩니다

## 빌드

Visual Studio 2022 Build Tools(C++ 데스크톱 개발 워크로드)와 CMake 필요.

```powershell
"C:\Program Files (x86)\Microsoft Visual Studio\2022\BuildTools\VC\Auxiliary\Build\vcvars64.bat"
cmake -S . -B build -G "NMake Makefiles"
cmake --build build
```

빌드 결과: `build\HyoExam.exe`. 실행 파일 옆에 `data\schedules.json`이 있어야 합니다
(CMake가 자동 복사하지 않으므로, 배포 시 `data\` 폴더를 exe와 함께 배치할 것).

## 데이터 파일

- `data/schedules.json` — 시험 유형별 교시/시간/과목/쉬는시간/안내문구.
  구조는 파일 내 `mock_exam`(모의고사), `midterm`(지필평가) 예시 참고. 학교마다
  시간표가 다르므로 각 학교에 맞게 직접 수정.
- `%APPDATA%\HyoExam\settings.json` — 테마·시간 기준·마지막 선택
  시험 유형 (앱이 자동 생성/저장).
- `%APPDATA%\HyoExam\profiles.json` — 툴바 저장/불러오기 기능으로 저장한
  이름별 설정 스냅샷(테마·시간 기준·시간표 전체).
- `%APPDATA%\HyoExam\memo.rtf` — 좌측 하단 메모 영역 내용(서식 포함, RTF).

## 브랜드 자산

`assets/gen_icon.py` (Pillow 필요)로 HyoT 브랜드 가이드에 맞는 아이콘을
재생성할 수 있습니다: 대각선 그라데이션(#4A9FE0→#2B7CC7), 흰색 단일 글리프,
투명 배경. 산출물: `assets/hyoexam.ico` (실행 파일 아이콘), `data/icon.webp`
(hyot.dev 등록용 512×512).

UI 폰트는 Windows 10/11 기본 내장 **맑은 고딕**으로 고정(별도 설치 불필요).
숫자/버전 표기만 브랜드 규정에 따라 JetBrains Mono를 사용합니다.
